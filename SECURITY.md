# Security Policy

## Supported Versions

CRUXpider is currently maintained on the latest `main` branch and the latest tagged release.

## Reporting a Vulnerability

Please do not open public issues for security-sensitive bugs.

Instead:

1. Email the maintainer directly if a contact address is available.
2. If email is not available, open a GitHub security advisory or private vulnerability report.
3. Include clear reproduction steps, impact, and any proposed mitigation.

## Scope

Security reports are especially useful for:

- secret leakage
- dependency vulnerabilities with practical impact
- authentication or authorization bugs
- unsafe file handling
- server-side request or injection issues

## Response Expectations

The project aims to acknowledge valid reports within 7 days and publish a fix or mitigation plan as soon as practical.
